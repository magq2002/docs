## Overview

This endpoint returns comprehensive details for a specific purchase order, including complete item information, delivery status, tracking updates, and order history.

<RequestExample>
```bash cURL
curl -X GET 'https://api.example.com/api/v2/collaborator/purchase-orders/ord_67890' \
  -H 'Authorization: Bearer YOUR_API_TOKEN'
```

```javascript Node.js
const response = await fetch('/api/v2/collaborator/purchase-orders/ord_67890', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${apiToken}`
  }
});
```

```python Python
import requests

response = requests.get(
  'https://api.example.com/api/v2/collaborator/purchase-orders/ord_67890',
  headers={
    'Authorization': f'Bearer {api_token}'
  }
)
```
</RequestExample>

## Parameters

<ParamField path="order_id" type="string" required>
  Unique identifier of the order to retrieve. Must be a valid order ID belonging to the authenticated collaborator.
</ParamField>

## Tracking Event Types

| Event Status | Description | Typical Timeline |
|--------------|-------------|------------------|
| `order_confirmed` | Order placed and payment processed | Day 0 |
| `processing` | Order being prepared for shipment | Days 1-2 |
| `shipped` | Package dispatched from warehouse | Day 2-3 |
| `in_transit` | Package in transit to destination | Days 3-6 |
| `out_for_delivery` | Package out for final delivery | Day 6-7 |
| `delivered` | Package successfully delivered | Day 7 |
| `delivery_attempted` | Delivery attempted but failed | Variable |
| `exception` | Delivery exception or delay | Variable |

## Order Actions

<AccordionGroup>
  <Accordion title="Cancellation Rules">
    Orders can be cancelled:
    - Within 2 hours of placement
    - Before status changes to "processing"
    - Points are fully refunded for cancelled orders
  </Accordion>
  
  <Accordion title="Modification Rules">
    Orders can be modified:
    - Within 1 hour of placement
    - Only delivery address can be changed
    - Product changes require cancellation and new order
  </Accordion>
  
  <Accordion title="Reorder Capability">
    Orders can be reordered if:
    - All products are still available
    - Collaborator has sufficient points
    - Products haven't been discontinued
  </Accordion>
</AccordionGroup>

## Best Practices

<Tip>
**Real-time Updates**: Tracking information is updated in real-time. Refresh order details periodically for active shipments.
</Tip>

<Note>
**Image Optimization**: Product images are served in multiple resolutions. Use appropriate image sizes for your UI.
</Note>

<Warning>
**Privacy**: Order details contain sensitive information. Ensure proper access controls and data protection.
</Warning>
