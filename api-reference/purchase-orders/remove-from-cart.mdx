## Overview

This endpoint allows collaborators to remove specific items from their shopping cart. You can remove individual cart items by their unique cart item ID.

<RequestExample>
```bash cURL
curl -X DELETE 'https://api.example.com/api/v2/collaborator/purchase-orders/shopping-cart/remove/item_001' \
  -H 'Authorization: Bearer YOUR_API_TOKEN'
```

```javascript Node.js
const response = await fetch('/api/v2/collaborator/purchase-orders/shopping-cart/remove/item_001', {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${apiToken}`
  }
});
```

```python Python
import requests

response = requests.delete(
  'https://api.example.com/api/v2/collaborator/purchase-orders/shopping-cart/remove/item_001',
  headers={
    'Authorization': f'Bearer {api_token}'
  }
)
```
</RequestExample>

## Parameters

<ParamField path="cart_item_id" type="string" required>
  Unique identifier of the cart item to remove. This ID is obtained from the shopping cart contents.
</ParamField>

## Alternative Endpoint: Remove by Product ID

You can also remove items by product ID using this alternative endpoint:

<RequestExample>
```bash cURL - Remove by Product ID
curl -X DELETE 'https://api.example.com/api/v2/collaborator/purchase-orders/shopping-cart/remove' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer YOUR_API_TOKEN' \
  -d '{
    "product_id": "prod_12345",
    "quantity": 1
  }'
```

```javascript Node.js - Remove by Product ID
const response = await fetch('/api/v2/collaborator/purchase-orders/shopping-cart/remove', {
  method: 'DELETE',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${apiToken}`
  },
  body: JSON.stringify({
    product_id: 'prod_12345',
    quantity: 1
  })
});
```
</RequestExample>

### Alternative Parameters

<ParamField body="product_id" type="string" required>
  Product identifier to remove from cart
</ParamField>

<ParamField body="quantity" type="integer">
  Number of units to remove. If not specified or greater than cart quantity, removes all units of this product.
</ParamField>

## System Behavior

<AccordionGroup>
  <Accordion title="Complete Item Removal">
    When removing by `cart_item_id` or when `quantity` equals or exceeds cart quantity:
    - Removes entire cart item entry
    - Returns `removed_item` in response
    - Updates cart totals accordingly
  </Accordion>
  
  <Accordion title="Partial Quantity Removal">
    When removing specific quantity less than cart quantity:
    - Reduces item quantity by specified amount
    - Returns `updated_item` in response
    - Maintains same `cart_item_id`
  </Accordion>
  
  <Accordion title="Empty Cart Result">
    When removing the last item from cart:
    - Cart summary shows all zeros
    - Special message indicates empty cart state
    - Cart session remains active for future additions
  </Accordion>
</AccordionGroup>

## Removal Methods Comparison

| Method | Use Case | Precision | Response |
|--------|----------|-----------|----------|
| By `cart_item_id` | Remove specific cart entry | Exact item removal | Always removes entire item |
| By `product_id` | Remove product units | Quantity-based | Can remove partial quantities |

## Best Practices

<Tip>
**Use Cart Item ID**: For precise removal operations, use the cart item ID method to avoid ambiguity when the same product appears multiple times.
</Tip>

<Note>
**Quantity Validation**: When removing by product ID with quantity, the system will remove all units if the specified quantity exceeds the cart quantity.
</Note>

<Warning>
**Concurrent Modifications**: If the cart is modified by another session between viewing and removing, the operation may fail. Always handle 404 errors gracefully.
</Warning>

## Error Handling

Common error scenarios and recommended handling:

| Error Code | Recommended Action |
|------------|-------------------|
| `CART_ITEM_NOT_FOUND` | Refresh cart contents and update UI |
| `PRODUCT_NOT_IN_CART` | Remove item from client-side cart display |
| `INVALID_QUANTITY` | Validate quantity input on client side |
